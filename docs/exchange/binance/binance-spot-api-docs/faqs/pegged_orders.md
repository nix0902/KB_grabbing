<h1>Pegged orders</h1>
<p><strong>Disclaimer</strong>:</p>
<ul>
<li>This explanation only applies to the SPOT Exchange.</li>
<li>The symbols and values used here are fictional and do not imply anything about the actual setup on the live exchange.</li>
<li>For simplicity, the examples in this document do not include commission.</li>
</ul>
<h2 class="anchor anchorWithStickyNavbar_fMI7" id="what-are-pegged-orders">What are pegged orders?<a class="hash-link" aria-label="Direct link to What are pegged orders?" title="Direct link to What are pegged orders?" href="/docs/binance-spot-api-docs/faqs/pegged_orders#what-are-pegged-orders">​</a></h2>
<p>Pegged orders are essentially <strong>limit orders</strong> with the price derived from the order book.</p>
<p>For example, instead of using a specific price (e.g. SELL 1 BTC for at least 100,000 USDC) you can send orders like “SELL 1 BTC at the best asking price” to queue your order after the orders on the book at the highest price, or “BUY 1 BTC for 100,000 USDT or best offer, IOC” to cherry-pick the sellers at the lowest price, and only that price.</p>
<p>Pegged orders offer a way for market makers to match the best price with minimal latency, while retail users can get quick fills at the best price with minimal slippage.</p>
<p>Pegged orders are also known as “best bid-offer” or BBO orders.</p>

<h2 class="anchor anchorWithStickyNavbar_fMI7" id="what-order-types-support-pegged-orders">What order types support pegged orders?<a class="hash-link" aria-label="Direct link to What order types support pegged orders?" title="Direct link to What order types support pegged orders?" href="/docs/binance-spot-api-docs/faqs/pegged_orders#what-order-types-support-pegged-orders">​</a></h2>
<p>All order types, with the exception of <code>MARKET</code> orders, are supported by this feature.</p>
<p>Since both <code>STOP_LOSS</code> and <code>TAKE_PROFIT</code> orders place a <code>MARKET</code> order once the stop condition is met, these order types cannot be pegged.</p>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="limit-orders">Limit orders<a class="hash-link" aria-label="Direct link to Limit orders" title="Direct link to Limit orders" href="/docs/binance-spot-api-docs/faqs/pegged_orders#limit-orders">​</a></h3>
<p>Pegged limit orders immediately enter the market at the current best price:</p>
<ul>
<li><code>LIMIT</code>
<ul>
<li>With <code>pegPriceType=PRIMARY_PEG</code> only <code>timeInForce=GTC</code> is allowed.</li>
</ul>
</li>
<li><code>LIMIT_MAKER</code>
<ul>
<li>Only <code>pegPriceType=PRIMARY_PEG</code> is allowed.</li>
</ul>
</li>
</ul>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="stop-limit-orders">Stop-limit orders<a class="hash-link" aria-label="Direct link to Stop-limit orders" title="Direct link to Stop-limit orders" href="/docs/binance-spot-api-docs/faqs/pegged_orders#stop-limit-orders">​</a></h3>
<p>Pegged stop-limit orders enter the market at the best price when price movement triggers the stop order (via stop price or trailing stop):</p>
<ul>
<li><code>STOP_LOSS_LIMIT</code></li>
<li><code>TAKE_PROFIT_LIMIT</code></li>
</ul>
<p>That is, stop orders use the best price at the time when they are triggered, which is different from the price when the stop order is placed. Only the limit price can be pegged, not the stop price.</p>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="oco">OCO<a class="hash-link" aria-label="Direct link to OCO" title="Direct link to OCO" href="/docs/binance-spot-api-docs/faqs/pegged_orders#oco">​</a></h3>
<p>OCO order lists may use peg instructions.</p>
<ul>
<li>Any order in OCO can be pegged: both above and below orders, or only one of them.</li>
<li>Pegged orders enter at the best price when they are placed on the book:
<ul>
<li><code>LIMIT_MAKER</code> order enters immediately at the current best price</li>
<li><code>STOP_LOSS_LIMIT</code> and <code>TAKE_PROFIT_LIMIT</code> enter at the best price when they are triggered</li>
</ul>
</li>
<li><code>STOP_LOSS</code> and <code>TAKE_PROFIT</code> orders cannot be pegged.</li>
</ul>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="oto-and-otoco">OTO and OTOCO<a class="hash-link" aria-label="Direct link to OTO and OTOCO" title="Direct link to OTO and OTOCO" href="/docs/binance-spot-api-docs/faqs/pegged_orders#oto-and-otoco">​</a></h3>
<p>OTO order lists may use peg instructions as well.</p>
<ul>
<li>Any order in OTO can be pegged: both working and pending orders, or only one of them.</li>
<li>Pegged working order enters immediately at the current best price.</li>
<li>Pegged pending limit order enters at the best price after the working order has been filled.</li>
<li>Pegged pending stop-limit order enters at the best price when it is triggered.</li>
</ul>
<p>OTOCO order lists may contain pegged orders as well, similar to OTO and OCO.</p>
<h2 class="anchor anchorWithStickyNavbar_fMI7" id="which-symbols-allow-pegged-orders">Which symbols allow pegged orders?<a class="hash-link" aria-label="Direct link to Which symbols allow pegged orders?" title="Direct link to Which symbols allow pegged orders?" href="/docs/binance-spot-api-docs/faqs/pegged_orders#which-symbols-allow-pegged-orders">​</a></h2>
<p>Please refer to Exchange Information requests and look for the field <code>pegInstructionsAllowed</code>. If set to true, pegged orders can be used with the symbol.</p>
<h2 class="anchor anchorWithStickyNavbar_fMI7" id="which-filters-are-applicable-to-pegged-orders">Which Filters are applicable to pegged orders?<a class="hash-link" aria-label="Direct link to Which Filters are applicable to pegged orders?" title="Direct link to Which Filters are applicable to pegged orders?" href="/docs/binance-spot-api-docs/faqs/pegged_orders#which-filters-are-applicable-to-pegged-orders">​</a></h2>
<p>Pegged orders are required to pass all applicable filters with the selected price:</p>
<ul>
<li><code>PRICE_FILTER</code></li>
<li><code>PERCENT_PRICE</code> and <code>PERCENT_PRICE_BY_SIDE</code></li>
<li><code>NOTIONAL</code> and <code>MIN_NOTIONAL</code> (considering the <code>quantity</code>)</li>
</ul>
<p>If a pegged order specifies <code>price</code>, it must pass validation at both <code>price</code> and <code>peggedPrice</code>.</p>
