<h1>Self Trade Prevention (STP) FAQ</h1>
<p><strong>Disclaimer:</strong></p>
<ul>
<li>The commissions and prices used here are fictional and do not imply anything about the actual setup on the live exchange.</li>
</ul>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="what-is-self-trade-prevention">What is Self Trade Prevention?<a class="hash-link" aria-label="Direct link to What is Self Trade Prevention?" title="Direct link to What is Self Trade Prevention?" href="/docs/binance-spot-api-docs/faqs/stp_faq#what-is-self-trade-prevention">​</a></h3>
<p>Self Trade Prevention (or STP) prevents orders of users, or the user&#x27;s <code>tradeGroupId</code> to match against their own.</p>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="what-defines-a-self-trade">What defines a self-trade?<a class="hash-link" aria-label="Direct link to What defines a self-trade?" title="Direct link to What defines a self-trade?" href="/docs/binance-spot-api-docs/faqs/stp_faq#what-defines-a-self-trade">​</a></h3>
<p>A self-trade can occur in either scenario:</p>
<ul>
<li>The order traded against the same account.</li>
<li>The order traded against an account with the same <code>tradeGroupId</code>.</li>
</ul>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="what-happens-when-stp-is-triggered">What happens when STP is triggered?<a class="hash-link" aria-label="Direct link to What happens when STP is triggered?" title="Direct link to What happens when STP is triggered?" href="/docs/binance-spot-api-docs/faqs/stp_faq#what-happens-when-stp-is-triggered">​</a></h3>
<p>There are five possible modes for what the system does when an order would create a self-trade.</p>
<p><code>NONE</code> - This mode exempts the order from self-trade prevention. Accounts or Trade group IDs will not be compared, no orders will be expired, and the trade will occur.</p>
<p><code>EXPIRE_TAKER</code> - This mode prevents a trade by immediately expiring the taker order&#x27;s remaining quantity.</p>
<p><code>EXPIRE_MAKER</code> - This mode prevents a trade by immediately expiring the potential maker order&#x27;s remaining quantity.</p>
<p><code>EXPIRE_BOTH</code> - This mode prevents a trade by immediately expiring both the taker and the potential maker orders&#x27; remaining quantities.</p>
<p><code>DECREMENT</code>  - This mode increases the <code>prevented quantity</code> of <em>both</em> orders by the amount of the prevented match. The smaller of the two orders will expire, or both if they have the same quantity.</p>
<p><code>TRANSFER</code> - If orders are from the same account, then the behavior is the same as <code>DECREMENT</code>.
If orders are from different accounts with the same <code>tradeGroupId</code>, then in addition to the behavior of <code>DECREMENT</code>, the <code>last prevented quantity</code> and its notional are transferred between the two accounts.</p>
<p>STP behavior is typically determined by the STP mode of the <strong>taker order</strong> only. The exception is that for STP <code>TRANSFER</code> to occur, both the maker and taker orders must specify STP mode <code>TRANSFER</code>. If the taker order specifies STP mode <code>TRANSFER</code>, but the maker order specifies a different STP mode, then the STP behavior is <code>DECREMENT</code>.</p>

<h3 class="anchor anchorWithStickyNavbar_fMI7" id="what-is-a-trade-group-id">What is a Trade Group Id?<a class="hash-link" aria-label="Direct link to What is a Trade Group Id?" title="Direct link to What is a Trade Group Id?" href="/docs/binance-spot-api-docs/faqs/stp_faq#what-is-a-trade-group-id">​</a></h3>
<p>Different accounts with the same <code>tradeGroupId</code> are considered part of the same &quot;trade group&quot;. Orders submitted by members of a trade group are eligible for STP according to the taker-order&#x27;s STP mode.</p>
<p>A user can confirm if their accounts are under the same <code>tradeGroupId</code> from the API either from <code>GET /api/v3/account</code> (REST API) or <code>account.status</code> (WebSocket API) for each account.</p>
<p>The field is also present in the response for <code>GET /api/v3/preventedMatches</code> (REST API) or <code>myPreventedMatches</code> (WebSocket API).</p>
<p>If the value is <code>-1</code>, then the <code>tradeGroupId</code> has not been set for that account, so the STP may only take place between orders of the same account.</p>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="what-is-a-prevented-match">What is a Prevented Match?<a class="hash-link" aria-label="Direct link to What is a Prevented Match?" title="Direct link to What is a Prevented Match?" href="/docs/binance-spot-api-docs/faqs/stp_faq#what-is-a-prevented-match">​</a></h3>
<p>When a self-trade is prevented, a prevented match is created. The orders in the prevented match have their prevented quantities increased and one or more orders expire.</p>
<p>This is not to be confused with a trade, as no orders will match.</p>
<p>This is a record of what orders could have self-traded.</p>
<p>This can be queried through the endpoint <code>GET /api/v3/preventedMatches</code> on the REST API or <code>myPreventedMatches</code> on the WebSocket API.</p>
