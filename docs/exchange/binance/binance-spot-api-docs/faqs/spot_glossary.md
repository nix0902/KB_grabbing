<h1>SPOT API Glossary</h1>
<p><strong>Disclaimer:</strong> This glossary refers only to the SPOT API Implementation. The definition for these terms may differ with regards to Futures, Options, and other APIs by Binance.</p>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="a">A<a class="hash-link" aria-label="Direct link to A" title="Direct link to A" href="/docs/binance-spot-api-docs/faqs/spot_glossary#a">​</a></h3>
<p><code>ACK</code></p>
<ul>
<li><code>newOrderRespType</code> enum. Stands for a type of order response in JSON where only the following fields are emitted: <code>symbol</code>, <code>orderId</code>, <code>orderListId</code>, <code>clientOrderId</code>, and <code>transactTime</code>.</li>
</ul>
<p><code>aggTrade</code>/Aggregate trade</p>
<ul>
<li>Aggregation of one or more individual trades from the same taker order that got filled at the same time and price.</li>
</ul>
<p>allocation</p>
<ul>
<li>Transfer of asset from the exchange to your account (e.g., when an order is filled by SOR instead of trading directly).</li>
</ul>
<p><code>allocationId</code></p>
<ul>
<li>Unique identifier of an allocation on a symbol.</li>
</ul>
<p><code>allocationType</code></p>
<ul>
<li>See <a href="/docs/binance-spot-api-docs/enums#allocationtype">AllocationType</a></li>
</ul>
<p><code>askPrice</code></p>
<ul>
<li>In ticker responses: the lowest price on the <code>SELL</code> side.</li>
</ul>
<p><code>askQty</code></p>
<ul>
<li>In ticker responses: total quantity offered at the lowest price on the <code>SELL</code> side.</li>
</ul>
<p><code>asks</code></p>
<ul>
<li>Orders on the <code>SELL</code> side.</li>
</ul>
<p><code>avgPrice</code></p>
<ul>
<li>Represents the volume-weighted average price for a set interval of minutes.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="b">B<a class="hash-link" aria-label="Direct link to B" title="Direct link to B" href="/docs/binance-spot-api-docs/faqs/spot_glossary#b">​</a></h3>
<p><code>baseAsset</code></p>
<ul>
<li>The first asset in the symbol (e.g. <code>BTC</code> is the <code>base asset</code> of symbol <code>BTCUSDT</code>), which represents the asset being bought and sold (the <code>quantity</code>).</li>
</ul>
<p><code>baseAssetPrecision</code></p>
<ul>
<li>A field found in Exchange Information that shows the number of decimals allowed on the <code>baseAsset</code>.</li>
</ul>
<p><code>baseCommissionPrecision</code></p>
<ul>
<li>A field found in Exchange Information that represents the number of decimals base asset commission will be calculated to.</li>
</ul>
<p><code>bidPrice</code></p>
<ul>
<li>In ticker responses: the highest price on the <code>BUY</code> side.</li>
</ul>
<p><code>bidQty</code></p>
<ul>
<li>In ticker responses: total quantity offered at the highest price on the <code>BUY</code> side.</li>
</ul>
<p><code>bids</code></p>
<ul>
<li>Orders on the <code>BUY</code> side.</li>
</ul>
<p><code>BREAK</code></p>
<ul>
<li>Symbol&#x27;s trading status that represents the symbol is not available for trading, which can happen during expected downtime. Market data is not generated during <code>BREAK</code>.</li>
</ul>
<p><code>BUY</code></p>
<ul>
<li>An enum in the <code>side</code> parameter when a user wants to purchase an asset (e.g. <code>BTC</code>).</li>
</ul>

<h3 class="anchor anchorWithStickyNavbar_fMI7" id="c">C<a class="hash-link" aria-label="Direct link to C" title="Direct link to C" href="/docs/binance-spot-api-docs/faqs/spot_glossary#c">​</a></h3>

</ul>
<p><code>clientOrderId</code></p>
<ul>

</ul>
<p><code>commission</code></p>
<ul>

</ul>
<p><code>commissionAsset</code></p>

</ul>
<p>Counter Order Id</p>

<p>Counter Symbol</p>

</ul>
<p><code>cummulativeQuoteQty</code></p>

</ul>

<h3 class="anchor anchorWithStickyNavbar_fMI7" id="d">D<a class="hash-link" aria-label="Direct link to D" title="Direct link to D" href="/docs/binance-spot-api-docs/faqs/spot_glossary#d">​</a></h3>
<p>Data Source</p>
<ul>

</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="e">E<a class="hash-link" aria-label="Direct link to E" title="Direct link to E" href="/docs/binance-spot-api-docs/faqs/spot_glossary#e">​</a></h3>
<p><code>executedQty</code></p>
<ul>
<li>The field that shows how much of the quantity was filled in an order.</li>
</ul>
<p><code>EXPIRED</code></p>
<ul>
<li>Order <code>status</code> indicating the order was canceled according to the order type&#x27;s rules or by the exchange.</li>
</ul>
<p><code>EXPIRED_IN_MATCH</code></p>
<ul>
<li>Order <code>status</code> indicating the order was canceled by the exchange due to STP. (e.g. an order with <code>EXPIRE_TAKER</code> will match with existing orders on the book with the same account or same <code>tradeGroupId</code>)</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="f">F<a class="hash-link" aria-label="Direct link to F" title="Direct link to F" href="/docs/binance-spot-api-docs/faqs/spot_glossary#f">​</a></h3>
<p><code>filters</code></p>
<ul>
<li>Defines the trading rules on the exchange.</li>
</ul>
<p><code>FOK</code>/ Fill or Kill</p>
<ul>
<li><code>timeInForce</code> enum where the order will not fill and expire if the order cannot be fully filled.</li>
</ul>
<p><code>free</code></p>
<ul>
<li>The amount of an asset in a user&#x27;s balances that can be used to trade or withdraw.</li>
</ul>
<p><code>FULL</code></p>
<ul>
<li><code>newOrderRespType</code> enum. Stands for a type of order response in JSON, where all the order information is emitted, including orders <code>fills</code> field.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="g">G<a class="hash-link" aria-label="Direct link to G" title="Direct link to G" href="/docs/binance-spot-api-docs/faqs/spot_glossary#g">​</a></h3>
<p><code>GTC</code>/ Good Til Canceled</p>
<ul>
<li><code>timeInForce</code> enum where the order will remain active until it is canceled or fully filled.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="h">H<a class="hash-link" aria-label="Direct link to H" title="Direct link to H" href="/docs/binance-spot-api-docs/faqs/spot_glossary#h">​</a></h3>
<p><code>HALT</code></p>
<ul>
<li>Symbol&#x27;s trading status that represents the symbol is not available for trading, which can happen during emergency downtime. Market data is still generated during <code>HALT</code>.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="i">I<a class="hash-link" aria-label="Direct link to I" title="Direct link to I" href="/docs/binance-spot-api-docs/faqs/spot_glossary#i">​</a></h3>
<p><code>intervalNum</code></p>
<ul>
<li>Describes the amount of time in the interval (e.g. if <code>interval</code> is <code>SECOND</code> and <code>intervalNum</code> is 5, then this will be interpreted as every 5 seconds).</li>
</ul>
<p><code>IOC</code> / Immediate or Canceled</p>
<ul>
<li><code>timeInForce</code> enum where the order tries to fill as much as possible, and the remaining unfilled quantity will expire.</li>
</ul>
<p><code>isBestMatch</code></p>
<ul>
<li>Field in the Response JSON that determines if the price of the trade was the best available on the exchange.</li>
</ul>
<p><code>isBuyerMaker</code></p>
<ul>
<li>Field in the Response JSON that indicates if the Buy side (the Buyer) was also the market maker (the Maker).</li>
</ul>
<p><code>isWorking</code></p>
<ul>
<li>Field in the JSON that shows if the order has started working on the order book.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="k">K<a class="hash-link" aria-label="Direct link to K" title="Direct link to K" href="/docs/binance-spot-api-docs/faqs/spot_glossary#k">​</a></h3>
<p><code>kline</code></p>
<ul>
<li>Identifies the open, close, high, low price, trading volume, and other market data, of a symbol at a specified time for a specific duration. Also known as a Candlestick.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="l">L<a class="hash-link" aria-label="Direct link to L" title="Direct link to L" href="/docs/binance-spot-api-docs/faqs/spot_glossary#l">​</a></h3>
<p>Last Prevented Quantity</p>
<ul>
<li>Order quantity that expired due to STP.</li>
</ul>
<p><code>lastPrice</code></p>
<ul>
<li>Price of the latest trade.</li>
</ul>
<p><code>lastQty</code></p>
<ul>
<li>Total quantity traded at the <code>lastPrice</code>.</li>
</ul>
<p><code>LIMIT</code></p>
<ul>
<li>a <code>type</code> of order where the execution price will be no worse than the order&#x27;s set price. The execution price is limited to be the set price or better.</li>
</ul>
<p><code>LIMIT_MAKER</code></p>
<ul>
<li>A <code>type</code> of order where the order can only be a maker order (i.e. The order cannot immediately match and take).</li>
</ul>
<p><code>limitClientOrderId</code></p>
<ul>
<li>A parameter used in placing OCO orders that identifies the <code>LIMIT_MAKER</code> pair of the OCO Order.</li>
</ul>
<p><code>listClientOrderId</code></p>
<ul>
<li>A parameter used in placing OCO Orders that identifies the pair of orders.</li>
</ul>
<p><code>locked</code></p>
<ul>
<li>The amount of an asset in a user&#x27;s balances that are currently locked in open orders and other services by the platform.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="m">M<a class="hash-link" aria-label="Direct link to M" title="Direct link to M" href="/docs/binance-spot-api-docs/faqs/spot_glossary#m">​</a></h3>
<p><code>MARKET</code></p>
<ul>
<li>A <code>type</code> of order where the user buys or sells an asset at the best available prices and liquidity until the order is fully filled or the order book&#x27;s liquidity is exhausted.</li>
</ul>
<p>Matching Engine</p>
<ul>
<li>This can either refer to a Data Source in the documentation which means the response is coming from the engine.</li>
<li>Or this is referred to as the system that handles all the requests and matches orders.</li>
</ul>
<p>Match Type</p>
<ul>
<li>Field in the order response or execution report indicating if the order was filled by the <a href="/docs/binance-spot-api-docs/faqs/sor_faq">SOR</a></li>
</ul>
<p>Memory</p>
<ul>
<li>Data Source where the response is coming from the API&#x27;s internal memory or cache.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="n">N<a class="hash-link" aria-label="Direct link to N" title="Direct link to N" href="/docs/binance-spot-api-docs/faqs/spot_glossary#n">​</a></h3>
<p><code>NEW</code></p>
<ul>
<li>Order <code>status</code> where a order has been successfully sent to the Matching Engine.</li>
</ul>
<p><code>newClientOrderId</code></p>
<ul>
<li>Parameter used in the SPOT API to assign the <code>clientOrderId</code> for the order being placed or the cancel message.</li>
</ul>
<p>Notional value</p>
<ul>
<li>The <code>price</code> * <code>qty</code> value.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="o">O<a class="hash-link" aria-label="Direct link to O" title="Direct link to O" href="/docs/binance-spot-api-docs/faqs/spot_glossary#o">​</a></h3>
<p><code>OCO</code></p>
<ul>
<li>One-Cancels-the-Other type of order that is composed by a pair of orders (e.g. <code>STOP_LOSS</code> or <code>STOP_LOSS_LIMIT</code> paired with a <code>LIMIT_MAKER</code> order) with the condition that if one of the orders execute, the other is automatically expired.</li>
</ul>
<p><code>OPO</code></p>
<ul>
<li><a href="https://github.com/binance/binance-spot-api-docs/blob/master/faqs/opo.md" target="_blank" rel="noopener noreferrer">One-Pays-The-Other</a>, a special subset of OTO.</li>
<li>When the working order fully fills, the accumulated received quantity is used for the quantity of the pending order.</li>
</ul>
<p><code>OPOCO</code></p>
<ul>
<li><a href="https://github.com/binance/binance-spot-api-docs/blob/master/faqs/opo.md" target="_blank" rel="noopener noreferrer">One-Pays-The-Other</a>, a special subset of OTOCO.</li>
<li>When the working order fully fills, the accumulated received quantity is used for the quantity of the pending OCO pair.</li>
</ul>
<p>Order Amend Keep Priority</p>
<ul>
<li>See <a href="/docs/binance-spot-api-docs/faqs/order_amend_keep_priority">Order Amend Keep Priority</a></li>
</ul>
<p>Order Book</p>
<ul>
<li>List of the open bids and asks for a symbol.</li>
</ul>
<p>Order List</p>
<ul>
<li>Multiple orders grouped together as a unit. See <code>OCO</code> and/or <code>OTO</code></li>
</ul>
<p><code>orderId</code></p>
<ul>
<li>A field in the order response that uniquely identifies the order on a symbol.</li>
</ul>
<p><code>origQty</code></p>
<ul>
<li>The original <code>quantity</code> that was sent during order placement.</li>
</ul>
<p><code>origClientOrderId</code></p>
<ul>
<li>Field used when canceling or querying an order by providing the <code>clientOrderId</code>.</li>
</ul>
<p><code>OTO</code></p>
<ul>
<li>One-Triggers-the-Other type of order that has a working order and a pending order.</li>
<li>When the working order fully fills, the pending order is automatically placed.</li>
</ul>
<p><code>OTOCO</code></p>
<ul>
<li>One-Triggers-a-One-Cancels-the-Other order has a working order, and an OCO pair for the pending orders.</li>
<li>When the working order fully fills, the pending OCO pair is automatically placed.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="p">P<a class="hash-link" aria-label="Direct link to P" title="Direct link to P" href="/docs/binance-spot-api-docs/faqs/spot_glossary#p">​</a></h3>
<p><code>PARTIALLY_FILLED</code></p>
<ul>
<li>Order <code>status</code> indicating that part of the order has been partially filled.</li>
</ul>
<p>Pending order</p>
<ul>
<li>An order in an order list that is only placed on the order book when the corresponding working order is fully filled.</li>
<li>A single order list can contain either a single pending order, or 2 pending orders forming an OCO.</li>
<li>In the single order case, almost any order type is supported, with the exception of <code>MARKET</code> orders using <code>quoteOrderQty</code>.</li>
</ul>
<p><code>PENDING_NEW</code></p>
<ul>
<li>Order <code>status</code> indicating that the pending orders of an order list have been accepted by the engine, but are not yet placed on the order book.</li>
</ul>
<p>Prevented execution price</p>
<ul>
<li>A field in User Data Stream execution reports showing the price of a prevented self-trade. See <a href="/docs/binance-spot-api-docs/faqs/stp_faq">STP</a>.</li>
</ul>
<p>Prevented execution quantity</p>
<ul>
<li>A field in the User Data Stream showing the quantity of a prevented self-trade. See <a href="/docs/binance-spot-api-docs/faqs/stp_faq">STP</a>.</li>
</ul>
<p>Prevented execution quote quantity</p>
<ul>
<li>A field in the User data Stream showing the quote quantity of the prevented self-trade. See <a href="/docs/binance-spot-api-docs/faqs/stp_faq">STP</a>.</li>
</ul>
<p><code>preventedQuantity</code></p>
<ul>
<li>Order quantity expired due to STP events.</li>
</ul>
<p>Prevented Match</p>
<ul>
<li>When order(s) expire due to the STP, a &quot;prevented match&quot; records the event.</li>
</ul>
<p><code>preventedMatchId</code></p>
<ul>
<li>When used in combination with <code>symbol</code>, can be used to query a prevented match of the expired order.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="q">Q<a class="hash-link" aria-label="Direct link to Q" title="Direct link to Q" href="/docs/binance-spot-api-docs/faqs/spot_glossary#q">​</a></h3>
<p><code>quantity</code></p>
<ul>
<li>Parameter used to specify the amount of the <code>base asset</code> to buy or sell.</li>
</ul>
<p><code>quoteAsset</code></p>
<ul>
<li>The second asset in the symbol (e.g. <code>USDT</code> is the <code>quote asset</code> of symbol <code>BTCUSDT</code>) which represents the asset being used to quote prices (the <code>price</code>).</li>
</ul>
<p><code>quoteAssetPrecision</code></p>
<ul>
<li>A field found in Exchange Information that shows the number of decimals allowed on the <code>quoteAsset</code>.</li>
</ul>
<p><code>quoteCommissionPrecision</code></p>
<ul>
<li>A field found in Exchange Information that represents the number of decimals quote asset commission will be calculated to.</li>
</ul>
<p><code>quoteOrderQty</code></p>
<ul>
<li><code>MARKET</code> order parameter that specifies the amount of the quote asset one wants to spend/receive in a &quot;Reverse MARKET order&quot;.</li>
</ul>
<p><code>quoteQty</code></p>
<ul>
<li><code>price</code> * <code>qty</code>; the notional value.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="r">R<a class="hash-link" aria-label="Direct link to R" title="Direct link to R" href="/docs/binance-spot-api-docs/faqs/spot_glossary#r">​</a></h3>
<p><code>recvWindow</code></p>
<ul>
<li>Parameter in the APIs that can be used to specify the number of milliseconds after the <code>timestamp</code> the request is valid for.</li>
</ul>
<p><code>RESULT</code></p>
<ul>
<li><code>newOrderRespType</code> enum. Stands for a type of order response in JSON, where all the order information is emitted, except order&#x27;s <code>fills</code> field.</li>
</ul>
<p>Reverse <code>MARKET</code> order</p>
<ul>
<li>A <code>MARKET</code> order that is specified using the <code>quoteOrderQty</code> instead of the <code>quantity</code>.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="s">S<a class="hash-link" aria-label="Direct link to S" title="Direct link to S" href="/docs/binance-spot-api-docs/faqs/spot_glossary#s">​</a></h3>
<p>Self Trade Prevention (STP)</p>
<ul>
<li>Self Trade Prevention is a feature that prevents orders of users, or the user&#x27;s <code>tradeGroupId</code> from matching against their own. Read <a href="/docs/binance-spot-api-docs/faqs/stp_faq">STP FAQ</a> to learn more.</li>
</ul>
<p><code>selfTradePreventionMode</code></p>
<ul>
<li>A parameter used to specify what the system will do if an order could cause a self-trade.</li>
</ul>
<p><code>SELL</code></p>
<ul>
<li>An enum in the <code>side</code> used when a user wants to sell an asset (e.g. BTC).</li>
</ul>
<p>Smart Order Routing (SOR)</p>
<ul>
<li>Smart Order Routing uses interchangeable quote assets to improve liquidity. Read <a href="/docs/binance-spot-api-docs/faqs/sor_faq">SOR FAQ</a> to learn more.</li>
</ul>
<p><code>specialCommissionForOrder</code>/<code>specialCommission</code></p>
<ul>
<li>See <a href="/docs/binance-spot-api-docs/faqs/commission_faq">Commission FAQ</a></li>
</ul>
<p><code>SPOT</code></p>
<ul>
<li>This is to distinguish a type of trading, where the purchase and delivery of a asset is made immediately.</li>
</ul>
<p><code>standardCommissionForOrder</code>/<code>standardCommission</code></p>
<ul>
<li>See <a href="/docs/binance-spot-api-docs/faqs/commission_faq">Commission FAQ</a></li>
</ul>
<p><code>stopClientOrderId</code></p>
<ul>
<li>A parameter used in placing OCO orders that identifies the <code>STOP_LOSS</code> or <code>STOP_LOSS_LIMIT</code> pair of the OCO Order.</li>
</ul>
<p><code>stopPrice</code></p>
<ul>
<li>The price used in algorithmic orders (e.g. <code>STOP_LOSS</code>, <code>TAKE_PROFIT</code>) that determines when an order will be triggered to be placed on the order book.</li>
<li>The price used in trailing algorithmic orders (e.g. <code>STOP_LOSS</code>, <code>TAKE_PROFIT</code>) to determine when trailing price tracking begins.</li>
</ul>
<p><code>STOP_LOSS</code></p>
<ul>
<li>A <code>type</code> of algorithmic order where once the market price hits the <code>stopPrice</code>, a <code>MARKET</code> order is placed on the order book.</li>
</ul>
<p><code>STOP_LOSS_LIMIT</code></p>
<ul>
<li>A <code>type</code> of algorithmic order where once the market price hits the <code>stopPrice</code>, a <code>LIMIT</code> order is placed on the order book.</li>
</ul>
<p><code>strategyId</code></p>
<ul>
<li>Arbitrary numeric value identifying the order within an order strategy.</li>
</ul>
<p><code>strategyType</code></p>
<ul>
<li>Arbitrary numeric value identifying the order strategy.</li>
</ul>
<p><code>symbol</code></p>
<ul>
<li>A trading pair, composed of a <code>base asset</code> and a <code>quote asset</code>. (e.g. BTCUSDT and BNBBTC)</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="t">T<a class="hash-link" aria-label="Direct link to T" title="Direct link to T" href="/docs/binance-spot-api-docs/faqs/spot_glossary#t">​</a></h3>
<p><code>TAKE_PROFIT</code></p>
<ul>
<li>A <code>type</code> of algorithmic order where once the market price hits the <code>stopPrice</code>, a <code>MARKET</code> order is placed on the order book.</li>
</ul>
<p><code>TAKE_PROFIT_LIMIT</code></p>
<ul>
<li>A <code>type</code> of algorithmic order where once the market price hits the <code>stopPrice</code>, a <code>LIMIT</code> order is placed on the order book.</li>
</ul>
<p><code>taxCommissionForOrder</code>/<code>taxCommission</code></p>
<ul>
<li>See <a href="/docs/binance-spot-api-docs/faqs/commission_faq">Commission FAQ</a></li>
</ul>
<p><code>ticker</code></p>
<ul>
<li>Reports the price change, and other maker data, of a symbol within a certain rolling interval.</li>
</ul>
<p><code>time</code></p>
<ul>
<li>For trade/allocation queries: the time when trades/allocations were executed.</li>
<li>For order queries: the time when orders were created.</li>
</ul>
<p><code>timeInForce</code></p>
<ul>
<li>Determines the taker behavior of an order, if an order can be a maker order, and how long the order will stay on the order book before it expires.</li>
<li>Supported enums are <code>GTC</code>, <code>IOC</code>, and <code>FOK</code>.</li>
</ul>
<p><code>tradeGroupId</code></p>
<ul>
<li>Group of accounts that belong to the same &quot;trade group&quot;.</li>
</ul>
<p><code>TRADING</code></p>
<ul>
<li>Trading status where orders can be placed.</li>
</ul>
<p><code>trailingDelta</code></p>
<ul>
<li>Trailing Stop Order parameter that specifies the delta price change required before order activation.</li>
</ul>
<p><code>trailingTime</code></p>
<ul>
<li>The time when the trailing order is now active and tracking price changes.</li>
</ul>
<p><code>transactTime</code></p>
<ul>
<li>The time when the order was updated: placed, filled, or canceled. This field (as well as all timestamp related fields) will be in milliseconds by default in JSON responses.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="u">U<a class="hash-link" aria-label="Direct link to U" title="Direct link to U" href="/docs/binance-spot-api-docs/faqs/spot_glossary#u">​</a></h3>
<p><code>uiKlines</code></p>
<ul>
<li>Modified candlestick data that is optimized for presentation of candlestick charts.</li>
</ul>
<p><code>updateTime</code></p>
<ul>
<li>Last update to the order. This field (as well as all timestamp related fields) will be in milliseconds by default in JSON responses.</li>
</ul>
<p>User Data Stream</p>
<ul>
<li>WebSocket stream used to get real-time information of a user&#x27;s account. (e.g. Changes to Balances, Order Updates, etc.) Read <a href="/docs/binance-spot-api-docs/user-data-stream">User Data Streams</a> to learn more.</li>
</ul>
<p><code>usedSor</code></p>
<ul>
<li>Indicates if the order was placed through <a href="/docs/binance-spot-api-docs/faqs/sor_faq">SOR</a>.</li>
</ul>
<hr>
<h3 class="anchor anchorWithStickyNavbar_fMI7" id="w">W<a class="hash-link" aria-label="Direct link to W" title="Direct link to W" href="/docs/binance-spot-api-docs/faqs/spot_glossary#w">​</a></h3>
<p><code>weightedAveragePrice</code></p>
<ul>
<li>The volume weighted average price in the last x minutes.</li>
</ul>
<p><code>workingFloor</code></p>
<ul>
<li>A field that determines whether the order is being filled by the SOR or by the order book the order was submitted to.</li>
</ul>
<p>Working order</p>
<ul>
<li>An order in an order list that is immediately placed on the order book, and will trigger the placement of one or multiple pending order(s) when it becomes fully filled.</li>
<li>An order in an order list that always consists of a single <code>LIMIT</code> or <code>LIMIT_MAKER</code> order.</li>
</ul>
<p><code>workingTime</code></p>
<ul>
<li>The time when the order started working on the order book.</li>
</ul>
