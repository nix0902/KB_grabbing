<h3 class="anchor anchorWithStickyNavbar_fMI7" id="status-codes">Status codes<a class="hash-link" aria-label="Direct link to Status codes" title="Direct link to Status codes" href="/docs/binance-spot-api-docs/testnet/websocket-api/response-format#status-codes">​</a></h3>
<p>Status codes in the <code>status</code> field are the same as in HTTP.</p>
<p>Here are some common status codes that you might encounter:</p>
<ul>
<li><code>200</code> indicates a successful response.</li>
<li><code>4XX</code> status codes indicate invalid requests; the issue is on your side.
<ul>
<li><code>400</code> – your request failed, see <code>error</code> for the reason.</li>
<li><code>403</code> – you have been blocked by the Web Application Firewall. This can indicate a rate limit violation or a security block. See <a href="https://www.binance.com/en/support/faq/detail/360004492232" target="_blank" rel="noopener noreferrer">https://www.binance.com/en/support/faq/detail/360004492232</a> for more details.</li>
<li><code>409</code> – your request partially failed but also partially succeeded, see <code>error</code> for details.</li>
<li><code>418</code> – you have been auto-banned for repeated violation of rate limits.</li>
<li><code>429</code> – you have exceeded API request rate limit, please slow down.</li>
</ul>
</li>
<li><code>5XX</code> status codes indicate internal errors; the issue is on Binance&#x27;s side.
<ul>
<li><strong>Important:</strong> If a response contains 5xx status code, it <strong>does not</strong> necessarily mean that your request has failed.
Execution status is <em>unknown</em> and the request might have actually succeeded.
Please use query methods to confirm the status.
You might also want to establish a new WebSocket connection for that.</li>
</ul>
</li>
</ul>
