<li>If you experience issues with the standard 443 port, alternative port 9443 is also available.</li>
</ul>
</li>
<li>A single connection to the API is only valid for 24 hours; expect to be disconnected after the 24-hour mark.</li>
<li>We support HMAC, RSA, and Ed25519 keys. For more information, please see <a href="/docs/binance-spot-api-docs/faqs/api_key_types">API Key types</a>.</li>
<li>Responses are in JSON by default. To receive responses in SBE, refer to the <a href="/docs/binance-spot-api-docs/faqs/sbe_faq">SBE FAQ</a> page.</li>
<li>If your request contains a symbol name containing non-ASCII characters, then the response may contain non-ASCII characters encoded in UTF-8.</li>
<li>Some methods may return asset and/or symbol names containing non-ASCII characters encoded in UTF-8 even if the request did not contain non-ASCII characters.</li>
<li>The WebSocket server will send a <code>ping frame</code> every 20 seconds.
<ul>
<li>If the WebSocket server does not receive a <code>pong frame</code> back from the connection within a minute the connection will be disconnected.</li>
<li>When you receive a ping, you must send a pong with a copy of ping&#x27;s payload as soon as possible.</li>
<li>Unsolicited <code>pong frames</code> are allowed, but will not prevent disconnection. <strong>It is recommended that the payload for these pong frames are empty.</strong></li>
</ul>
</li>
<li>Data is returned in <strong>chronological order</strong>, unless noted otherwise.
<ul>
<li>Without <code>startTime</code> or <code>endTime</code>, returns the most recent items up to the limit.</li>
<li>With <code>startTime</code>, returns oldest items from <code>startTime</code> up to the limit.</li>
<li>With <code>endTime</code>, returns most recent items up to <code>endTime</code> and the limit.</li>
<li>With both, behaves like <code>startTime</code> but does not exceed <code>endTime</code>.</li>
</ul>
</li>
<li>All timestamps in the JSON responses are in <strong>milliseconds in UTC by default</strong>. To receive the information in microseconds, please add the parameter <code>timeUnit=MICROSECOND</code> or <code>timeUnit=microsecond</code> in the URL.</li>
<li>Timestamp parameters (e.g. <code>startTime</code>, <code>endTime</code>, <code>timestamp</code>) can be passed in milliseconds or microseconds.</li>
<li>All field names and values are <strong>case-sensitive</strong>, unless noted otherwise.</li>
<li>If there are enums or terms you want clarification on, please see <a href="/docs/binance-spot-api-docs/faqs/spot_glossary">SPOT Glossary</a> for more information.</li>
<li>APIs have a timeout of 10 seconds when processing a request. If a response from the Matching Engine takes longer than this, the API responds with &quot;Timeout waiting for response from backend server. Send status unknown; execution status unknown.&quot; <a href="/docs/binance-spot-api-docs/testnet/errors#-1007-timeout">(-1007 TIMEOUT)</a>
<ul>
<li>This does not always mean that the request failed in the Matching Engine.</li>
<li>If the status of the request has not appeared in <a href="/docs/binance-spot-api-docs/testnet/user-data-stream">User Data Stream</a>, please perform an API query for its status.</li>
</ul>
</li>
<li><strong>Please avoid SQL keywords in requests</strong> as they may trigger a security block by a WAF (Web Application Firewall) rule. See <a href="https://www.binance.com/en/support/faq/detail/360004492232" target="_blank" rel="noopener noreferrer">https://www.binance.com/en/support/faq/detail/360004492232</a> for more details.</li>
