# Portfolio Settings 

#### How to open the settings menu 
 The initial portfolio settings are defined by the user during the creation of a new portfolio in the creation dialog.
 After the portfolio has been created, most of these parameters can be modified via the additional menu on the portfolio card on the All Portfolios page.

#### Settings parameters 

- Portfolio name — name of the portfolio. You can enter up to 128 characters. Portfolio names may be identical.
- Currency — portfolio currency. All asset values will be automatically converted into this currency if they are traded in another one. Cash holdings are also accounted for in the portfolio currency. Cannot be changed after the portfolio is created.
- Risk-free rate — the risk-free rate of return. This is the theoretical annual return that can be earned with zero or minimal risk. Typically, it may be based on bank deposit rates, the key interest rate of the main investment country, or long-term government bonds.[](https://www.investopedia.com/terms/r/risk-freerate.asp) It is used in the calculations of the [Sortino](https://www.tradingview.com/support/solutions/43000756110/) and [Sharpe](https://www.tradingview.com/support/solutions/43000756107/) ratios.
- Benchmark — market performance benchmark. Allows you to compare your portfolio's performance against a market reference. We recommend using an index or a fund corresponding to your portfolio's market or sector as a benchmark. Learn more in the related [article](https://www.tradingview.com/support/solutions/43000756149/).
- Description (optional) — portfolio description. This field may be left empty. Maximum length — 1200 characters.
- Auto adjust trades for splits — when this option is enabled, all transactions in your portfolio are automatically adjusted for stock splits. This ensures that your holdings and performance remain accurate after a split occurs. If disabled, splits will not be applied, and your portfolio will be calculated using the original transaction prices. Learn more in the related [article](https://www.tradingview.com/support/solutions/43000765406/).

#### Portfolio recalculation with updated parameters 

After settings are changed, the portfolio is automatically recalculated using the new values.The results of the recalculation may not appear immediately — this depends on the number of instruments and transactions in the portfolio. While the recalculation is in progress, the interface will display an active processing indicator — “Updating your data…”

The image shows a **status notification banner** from the TradingView trading platform. Here’s a detailed breakdown:  


### 1. Core UI Element: The Notification Banner  
- **Appearance**: A dark gray (near-black) rectangular box with rounded corners, containing text and a loading indicator.  
- **Text**: “Updating your data” (in white, sans-serif font) – this is the primary message.  
- **Loading Indicator**: Three small, evenly spaced dots (ellipsis) to the left of the text, which animate (or appear static here) to signal an ongoing process.  


### 2. Purpose of the Banner  
This banner is a **system status indicator** that informs the user:  
- The platform is actively refreshing or updating market data (e.g., price quotes, charts, order books, or other real-time information).  
- The user should wait for the update to complete before relying on the latest data (e.g., making trading decisions).  


### 3. Context in TradingView  
TradingView is a financial charting and analysis platform. This banner typically appears during:  
- Initial data load (when opening a chart or workspace).  
- Periodic data refreshes (e.g., real-time price updates, tick data, or historical data syncs).  
- Background processes (e.g., recalculating indicators, updating watchlists, or syncing with broker accounts).  


### 4. Design & Usability  
- **Visibility**: The dark background and white text ensure high contrast, making the notification easy to read.  
- **Non-intrusive**: It occupies minimal space (a small horizontal bar) and doesn’t block critical UI elements (e.g., charts, toolbars).  
- **Feedback**: The ellipsis clearly communicates “in progress” (vs. a static message), reducing user confusion about whether the update is stuck.  


In short, this is a standard “loading” or “syncing” notification in TradingView, designed to keep users informed about data updates without disrupting their workflow.
