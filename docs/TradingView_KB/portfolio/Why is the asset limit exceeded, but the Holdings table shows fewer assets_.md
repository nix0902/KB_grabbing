# Why is the asset limit exceeded, but the Holdings table shows fewer assets?

This situation is explained by the fact that the Holdings table, by default, displays only assets with open positions. However, the system counts the total number of unique assets that have ever been present in the portfolio, including those already sold. This total number is what is compared against the set limit. Additionally, make sure there are no transactions in the portfolio that are not displayed in the Holdings table due to their timing. Transactions added before the trading for the instrument has opened on the current date will not appear in the table until trading begins, but they are still counted toward the total number of unique assets. For more information, see the article: [Why is my transaction not included in the calculation?](https://www.tradingview.com/support/solutions/43000756241/) Solution: To display the full list of assets, including closed positions, activate the Display sold toggle in the Holdings table. If the issue persists after following these steps, we recommend contacting support for a detailed review of the situation.
