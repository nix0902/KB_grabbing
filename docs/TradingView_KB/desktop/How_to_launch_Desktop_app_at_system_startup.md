# How to launch Desktop app at system startup 
 On Windows, open the Start menu, launch the Settings app, and go to Apps. Open Startup, find the TradingView app on the list and switch the toggle on:

On macOS, right-click the app icon in the Dock and check “Open at Login” in Options:

**Описание:**

The image shows a **macOS Dock** (the horizontal bar of application icons at the bottom of the screen) with a **context menu** open, triggered by right-clicking (or secondary-clicking) an application icon. The menu is related to the application’s dock settings, and the visible elements are:  


### 1. **Dock & Application Icons**  
The Dock contains several app icons (from left to right):  
- A gear/cog icon (likely a system utility or custom app).  
- A black square with “TV” (possibly a media or TV-related app).  
- Google Chrome (colorful circular icon).  
- A keychain/lock icon (macOS Keychain Access).  
- A jar with a beach scene (possibly a custom or third-party app).  
- A gray trash can (macOS Trash).  


### 2. **Context Menu (Dock Settings)**  
The menu has two sections:  

#### Left Section (Primary Menu):  
- **“Options”** (with a right arrow): A submenu for additional dock-related settings (e.g., “Show in Dock,” “Remove from Dock”).  
- **“Show All Windows”**: A command to reveal all open windows of the selected app (e.g., if the app has multiple windows, this brings them to the foreground).  
- **“Show”**: A submenu (not expanded here) for showing the app’s windows or icon in the Dock.  
- **“Quit”**: Closes the selected application.  


#### Right Section (Submenu from “Options”):  
This submenu contains three dock preferences:  
- **“Keep in Dock”** (checked with a ✔️): Ensures the app’s icon remains in the Dock even after closing (prevents it from being removed automatically).  
- **“Open at Login”** (checked with a ✔️, highlighted in blue): Configures the app to launch automatically when the Mac starts up.  
- **“Show in Finder”**: Opens the app’s folder in Finder (to access its files or move it).  


### Purpose of UI Elements  
- **Dock**: Provides quick access to frequently used apps.  
- **Context Menu**: Manages app behavior in the Dock (persistence, startup, visibility, and quitting).  
- **“Keep in Dock”**: Retains the app’s icon in the Dock.  
- **“Open at Login”**: Automates app launch on system startup.  
- **“Show in Finder”**: Locates the app’s file system location.  
- **“Show All Windows”**: Organizes the app’s open windows.  
- **“Quit”**: Exits the app.  


This interface is typical of macOS, where right-clicking a Dock icon reveals options to customize app behavior and interaction with the Dock.
